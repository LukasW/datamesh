Das dbt SQL dahinter (5 Zeilen)
SELECT
    pr.product_line,    pr.product_name,
    COUNT(f.policy_id)          AS active_policies,
    SUM(f.premium_chf)          AS total_premium_chf,
    ROUND(AVG(f.premium_chf), 2) AS avg_premium_chf
FROM fact_policies f                        -- Policy domain data product
JOIN dim_product pr ON f.product_id = pr.product_id  -- Product domain data product
WHERE f.policy_status = 'ACTIVE'
GROUP BY pr.product_line, pr.product_name
Datenpipeline (nur Kafka — kein direkter DB-Zugriff)
policy_db (Data Inside) → Outbox + Debezium → policy.v1.issued (Data Outside) → raw.policy_events → stg_policy_events (dbt) → fact_policies (dbt) → mart_portfolio_summary (dbt)
Prinzip: Data Inside vs. Data Outside
Die operativen Datenbanken (partner_db, policy_db, product_db) sind Data Inside — sie wurden in diesem Query nie berührt. Alle Daten kamen über Kafka-Topics (Data Outside), die durch Open Data Contracts und Avro-Schemas versioniert sind.
